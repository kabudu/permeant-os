# AWS GPU E2E cross-host run - June 15, 2026

This note captures the first successful GPU-backed AWS cross-host run after the `us-east-1` EC2 `G and VT` quota propagated.

## Outcome

Verdict:
- PermeantOS completed a real cross-host migration from the live local MLX source host to a disposable AWS GPU instance.
- The live source side extracted tensors from the local MLX exporter over the command-backed adapter seam.
- The AWS target accepted the capability exchange, verified the encrypted header, streamed all payload blocks, and committed the migration successfully.
- All AWS resources created for the run were torn down afterward.

What this proves:
- live MLX source-process extraction
- cross-host migration from the laptop to a GPU-backed public-cloud target
- target-side header verification, block streaming, staged import, and commit
- full launch, run, teardown, and leftover-verification discipline on AWS

What this does not yet prove:
- direct registration into a production live target inference runtime
- continuation generation from a true live target runtime

## Environment summary

Source host:
- OS: macOS
- Architecture: aarch64
- Runtime exposure: local MLX exporter on `127.0.0.1:29101`

Target host:
- Provider: AWS EC2
- Instance type: `g4dn.xlarge`
- Region: `us-east-1`
- Availability Zone: `us-east-1d`
- OS: Amazon Linux 2023 x86_64

Transport path:
- local SSH tunnel from the laptop to the EC2 host
- daemon remained bound to `127.0.0.1:29099` on the target host

Operational note:
- the stock Amazon Linux image on this host did not have `nvidia-smi` installed
- GPU-backed execution was verified by the successful `g4dn.xlarge` launch and the successful migration/commit path
