# PermeantOS v0.1.0

PermeantOS v0.1.0 is the first official product release of the open-source
agent-state migration platform.

The headline is simple: agents can move.

This release packages the work that turns PermeantOS from a research preview
into a usable, installable project with published Rust crates, signed release
artifacts, production-release gates, and evidence-backed runtime migration
claims.

## What Is Included

- Live agent-state migration primitives for moving runtime state between
  compatible AI runtime profiles.
- Agent Memory Graph schema and harness support for binding memory graph
  activity to migration evidence.
- Production transport foundations, including secure WebSocket transport work
  and fallback-aware transport planning.
- Runtime adapter boundaries for extraction, normalisation, injection, and
  orchestration.
- MLX-to-vLLM real-runtime evidence, including AWS-backed migrations with exact
  continuation fidelity, reverse export/import, return-home continuation, graph
  activity, and cleanup verification.
- llama.cpp proof work showing imported runtime state continuation and raw KV
  tensor write paths.
- QATQ-backed live migration integration using the published `qatq` crate,
  with recorded exact restoration and live compression-gate evidence against
  raw, `zstd`, and `lz4` baselines.
- A repeatable evidence index and validation reports that tie claims to
  commands, artifacts, checks, and known limitations.
- `permeant-cli starter-demo`, a one-command installed-binary smoke path for
  validating the release artifact locally.
- Guarded real-release automation for crates.io publication, signed/notarised
  macOS artifacts, Linux artifacts, checksums, manifests, and GitHub Releases.

## Published Crates

The v0.1.0 Rust workspace crates are published on crates.io:

- `usxf-core`
- `permeant-transport`
- `permeant-transpiler`
- `permeant-extractor`
- `permeant-injector`

The Python SDK remains intentionally unpublished in this release while the
crate and binary release path is hardened first.

- Release manifests for both platform artifacts.

## Evidence And Scope

This release makes a precise claim: PermeantOS has validated that agent runtime
state can move across the evaluated runtime/model profiles while preserving
state integrity and continuation fidelity.

The strongest current evidence includes:

- real AWS MLX-to-vLLM migrations;
- exact post-migration continuation checks;
- reverse export/import and origin return-home continuation;
- Agent Memory Graph activity attached to migrated execution;
- QATQ live migration compression evidence using the published `qatq` crate;
- llama.cpp target-state and raw KV write proof work.

PermeantOS is not claiming universal runtime portability yet. Broader runtime
coverage, longer-horizon automated evidence jobs, richer adapter conformance,
and additional production hardening are the next layer of the platform.

## Why This Matters

PermeantOS is building toward a world where agent state is portable, inspectable,
and resumable across runtimes and hardware profiles. v0.1.0 is the first
installable release that backs that idea with code, crates, binaries, signed
artifacts, and real migration evidence.
